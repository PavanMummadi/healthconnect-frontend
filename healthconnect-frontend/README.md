🏥 Hospital Management System – Frontend (React + Vite)
📌 Overview

A React + Vite–based frontend application for the Hospital Management System that provides a clean, responsive, and role-based user interface for Patients, Doctors, and Admins.
The application consumes secure REST APIs from the Spring Boot backend and enables online appointment booking, QR-based ticket validation using device camera, and prescription management.



🎯 Key Goals

Provide a smooth and intuitive hospital booking experience

Enable real-time interaction with backend APIs

Support role-based UI for Admin, Doctor, and User

Implement real-world hospital workflows digitally

👥 Roles

Admin – Manages doctors and system access

Doctor – Validates tickets, views patient history, adds prescriptions

User (Patient) – Books appointments, views tickets & prescriptions

🔐 Security

JWT-based authentication handling

Role-based route protection

Secure communication with backend APIs

✨ Core Features

User registration & login UI

Role-based dashboards

Doctor availability display

Appointment booking interface

QR code display for appointments

Camera-based QR code scanning for ticket validation

Doctor-specific ticket enforcement

Prescription viewing for users

Patient medical history visualization

Waiting list status handling

📷 QR Ticket Handling

Displays QR code generated by backend

Doctor scans QR using device camera

Ticket validated against assigned doctor

Invalid ticket handling for wrong doctor visits

📝 User Interaction Flow

User logs in and books appointment

QR ticket displayed in user dashboard

Doctor scans QR ticket via camera

Ticket validation performed

Prescription added after consultation

Prescription visible in user dashboard

Patient history preserved for future visits

🧩 Tech Stack

Frontend: React

Build Tool: Vite

Routing: React Router

API Calls: Axios

State Management: React Hooks

QR Scanning: Camera-based QR scanner

Styling: CSS / Tailwind / Bootstrap
